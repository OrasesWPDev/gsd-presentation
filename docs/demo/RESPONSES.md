# GSD Questioning Responses

**For:** Pomodoro Timer Demo
**Style:** Terse, direct answers. Let GSD do the talking.

---

## Usage

During `/gsd:new-project`, GSD will ask questions to understand your project. These are the pre-written responses for the Pomodoro timer demo.

Keep answers short. GSD will elaborate and ask follow-ups.

---

## Responses

**Q: What are you building?**
A: "Pomodoro timer"

**Q: Tell me more about the Pomodoro timer**
A: "25-min work sessions, 5-min breaks. Single HTML file. No dependencies."

**Q: Who is this for?**
A: "Me. Personal productivity."

**Q: What does done look like?**
A: "Timer counts down 25:00 to 00:00. Start/stop/reset. Shows completed sessions."

**Q: Any constraints?**
A: "Single HTML file. No frameworks. Runs locally in browser."

**Q: Ready to create PROJECT.md?**
A: Select "Create PROJECT.md"

---

## Additional Questions (if asked)

**Q: What tech stack?**
A: "Vanilla JS, HTML, CSS. No build tools."

**Q: Any specific design requirements?**
A: "Minimal. Dark background, light text. Monospace timer display."

**Q: Should it make sounds?**
A: "No audio. Visual only."

**Q: Multiple timer modes?**
A: "Yes. 25-min work, 5-min short break, 15-min long break."

**Q: Auto-start breaks?**
A: "Yes. Timer auto-switches mode when done."

---

## Response Style Guide

- Use 1-5 words when selecting from options
- Keep free-text answers under 20 words
- No justifications ("because..." or "I think...")
- No questions back to GSD
- Match option text exactly when available
